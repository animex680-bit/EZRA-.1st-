# Review Checklist
## Pre-Export Quality Check — Run Through This Before Every Delivery

---

## Instructions

Work through this list in order. Every item is a potential failure point.  
If an item fails, fix it before continuing. Don't skip items just because  
you "think it's probably fine."

Mark each item: [PASS] / [FAIL — FIXED] / [N/A for this project]

---

## Section A — Visual Foundation

```
[ ] Cinematic bars are on, locked, full opacity, exactly same height top and bottom
[ ] Grain overlay is active — Blend Mode: Overlay, Opacity: 15-25%, looping correctly
[ ] Color grade adjustment layer is active and applied correctly
[ ] LUT is applied at the correct strength (50-70% opacity)
[ ] No banding or compression artifacts visible in dark areas
[ ] No clipping (blown-out white areas) in highlights — check the subject's face
[ ] Background is dark at all times — no accidentally bright frames
[ ] Vignette is present (if used) — not too strong, not too light
```

---

## Section B — Beat Sync

```
[ ] Every cut lands within 2 frames of a beat marker
[ ] Every text entrance resolves on a beat (started 3-4 frames before)
[ ] The Twixtor snap / slow-mo release happens on the designated DROP beat
[ ] Every SFX lines up with its visual event (no SFX playing alone)
[ ] No visible "dead zone" longer than 3 seconds without text, graphics, or motion
[ ] The edit starts with impact on the first beat (no dead intro)
```

---

## Section C — Typography

```
[ ] All text is using fonts from the approved stack (Bebas Neue, Inter, Space Mono)
[ ] No Arial, Calibri, or default system fonts anywhere
[ ] Headline text is at correct size (100-160px for main statements)
[ ] Text is readable at 375px wide (phone screen) — zoom out and check
[ ] No text is cut off by the cinematic bars at top/bottom
[ ] Text is not overlapping the subject's face in an unintentional way
[ ] All text animations are enabled Motion Blur
[ ] No typos — read every word carefully
[ ] All lyrics match the actual audio lyrics (if lyric video)
[ ] Account handle / CTA present at or near the end
```

---

## Section D — Gradient Character Treatment

```
[ ] Gradient treatment is applied to the primary subject
[ ] Roto brush edges are clean — no jagged pixel edges visible
[ ] Gradient is animating (not static)
[ ] Glow effect is present — not too strong, not invisible
[ ] The gradient color matches the approved palette
[ ] Subject is distinguishable from background (not blending in)
```

---

## Section E — Speed Ramp

```
[ ] Twixtor is applied to the correct clip
[ ] Slow-motion section is smooth — no visible stuttering or ghosting
[ ] Speed ramp timing: slow-mo release is on the DROP beat
[ ] Camera shake is present on the snap-back (not before, not after)
[ ] The footage source was 60fps+ for the Twixtor clip (check if not, note degradation)
```

---

## Section F — Transitions

```
[ ] All transitions are intentional — no accidental hard cuts mid-word
[ ] Transition type matches the energy level of the moment
[ ] No cross-dissolves used (unless specifically requested — they break the style)
[ ] Flash frames at cuts are 1-2 frames only (not too long)
[ ] All transitions land on beats
```

---

## Section G — Audio

```
[ ] Music is audible at full volume, no clipping in the waveform
[ ] Integrated loudness is approximately -14 LUFS (measure with loudness meter)
[ ] True Peak is below -1 dBTP
[ ] SFX are present at key moments (impact, transitions, text hits)
[ ] SFX volumes are below the music at all times
[ ] No audio pops, clicks, or gaps in the music
[ ] Audio matches video sync — no drift at the end of the edit
```

---

## Section H — Shape Graphics

```
[ ] Shape layer graphics are present and visible (not accidentally hidden)
[ ] Shapes animate on beats, not randomly
[ ] Shape layer opacity is appropriate (not too opaque, not invisible)
[ ] No shape elements covering the subject's face or key action
[ ] Shapes are consistent with the edit's visual style (thin lines, not chunky)
```

---

## Section I — Technical

```
[ ] Composition settings are 1080 × 1920, 24fps (or 30fps if specified)
[ ] All pre-comps are rendering correctly (no purple/pink placeholder frames)
[ ] Motion Blur is enabled on the composition (Composition → Composition Settings → Motion Blur)
[ ] All footage proxies are linked to their originals (or originals are used directly)
[ ] No missing footage placeholders (black/colored boxes in timeline)
[ ] Project renders without errors in AE
[ ] Final duration matches the music (no extra seconds of silence at end)
```

---

## Section J — Final Export Verification

```
[ ] Export format: H.264 MP4
[ ] Resolution: 1080 × 1920
[ ] Frame rate matches composition
[ ] Bitrate: 10-20 Mbps (check file size — roughly 10-20MB per minute is typical)
[ ] Audio codec: AAC, 320kbps
[ ] File plays back correctly in VLC / QuickTime / phone
[ ] Colors look correct on a phone screen (not just on calibrated monitor)
[ ] Check the file on Instagram: upload as a DRAFT and preview before publishing
```
