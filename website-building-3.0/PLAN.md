# PLAN.md — The Master Plan for Excellent, High-Value Websites

Built 2026-06-12 from the analyzed workflows of 14 top creators (7 Instagram, 7
YouTube — `research/CREATORS.md`). This plan is **adaptive by design**: every "no"
from the user mutates it through the ADAPT.md protocol. It is a living document —
the version in git history is the past; the rules referenced here are the present.

---

## 1. The thesis (what the data says)

Three facts emerged from the research, and the whole plan stands on them:

1. **The lane is open.** AI speed-run creators (Riley Brown tier, 300K+ followers)
   ship generic sites fast. Premium-WebGL craftsmen (Mancini, Snellenberg) don't
   run public AI pipelines. The combination — **an AI-agent pipeline producing real
   WebGL signature work** — is nearly unoccupied, and validated at the very top:
   Niccolò Miranda's Revelium runs Codex + Claude Code + Cursor over three.js/SDF/
   WebGPU for Prada, Meta, Bvlgari. That is this repo's exact architecture.
2. **Process beats talent at scale.** Every top creator runs the same spine
   (reference → tokens → Blender assets → scroll choreography → signature shader →
   perf gate → ship → productize). Excellence is the spine executed without
   skipping steps — which is what phase gates are for.
3. **Quality = direction × revision rounds.** Verified across every AI workflow
   studied: the agent's first output is a draft; premium takes 2–3 *specifically
   directed* revision rounds. The plan budgets them instead of pretending one-shot.

## 2. The design doctrine (what makes a site "excellent high value")

Derived from the creator data, enforced by the anti-slop bar and the logic-check agent:

- **One signature moment** the site is *about* — a custom shader or interaction a
  template cannot produce (refraction material, GPGPU particle morph, scroll-
  scrubbed reveal). One perfected effect beats five mediocre ones.
- **Scroll is the spine.** The scroll position is the timeline: it creates the
  words, scrubs the media, drives the camera. No travel — choreography.
- **Nothing is static.** Tiles drift and react to the cursor; buttons glow and
  attract; sections blend through seamless blips, never hard edges.
- **Type is the hero.** Characterful editorial pairing from the tokens file —
  bland type sinks everything (both the research and the user's logged taste agree).
- **Brand system before generation.** Tokens-in, brand-out — the documented fix
  for AI-generic output.
- **Right-resolution assets only.** 2K–4K source material requested with precise
  specs, free CC0 sourced, or procedurally authored — never stretched, never faked.
- **It must SELL.** Proof architecture (case studies with price + fit), one clear
  repeated CTA, value proposition readable in 5 seconds. conversion-critic enforces.
- **It must RENDER.** Deterministic output (verified WebGL or video-driven page),
  error boundaries, reduced-motion path, perf budgets. logic-check + perf enforce.

## 3. How a site gets built (the short version)

`WORKFLOW.md` is the full runbook. Compressed:

```
RESEARCH (if stale) → BRIEF (1 page, user-confirmed) → CONCEPT [gate: sign-off]
→ ASSETS via asset-scout: free CC0 / procedural / 2K–4K user ask [gate: all routed]
→ BUILD (tokens first, stack per web3d, debug UI day one)
→ MOTION (the system, not effects) → QA [gate: logic-check SHIP + conversion-critic
+ perf] → DELIVER (live URL, always) → FEEDBACK (adapt loop) → EVOLVE (skill edits)
```

With 2–3 directed revision rounds budgeted inside BUILD→QA, every time.

## 4. The adaptation mechanism (why this plan can't go stale)

- **Taste:** every rejection rewrites a rule (`ADAPT.md`); the taste profile is the
  user's accumulating veto record, and it **outranks research trends**.
- **Craft:** every project appends LEARNINGS and edits skills (`web3d-evolve`).
- **Market:** every ~6 weeks, trend-scouts re-analyze the top creators 50/50
  IG/YouTube and the findings must change skill files (`wb3-research`).
- **QA:** every miss the user catches becomes a permanent logic-check rule —
  the same mistake cannot reach the user twice.

Four feedback loops, four time scales: per-feedback, per-project, per-month, per-miss.

## 5. The business plan (results, not just sites)

From the research money-map (`PATTERNS.md` §5) + web3d-business:

- **Now ($3–8k tier):** video-driven + motion-system sites for SMBs with money and
  weak digital (the AluCape outbound model: build the concept, DM the live link).
  Every delivery: case study with price + fit, ask for referral.
- **Next ($15–40k tier):** 3 flagship WebGL pieces that clear the anti-slop bar →
  submit to Awwwards. At the client tier, **one SOTD outweighs 100K followers** —
  prestige, not reach, is the trust signal.
- **Flywheel:** each build's techniques become 5–15s content units (micro-
  interaction close-ups, before/after, process BTS — the formats the IG data says
  win); building blocks get productized Osmo-style (snippets/components/skills as
  recurring revenue) once there are 3+ proven pieces.
- **Positioning:** sell the craft and the business result; the AI pipeline is the
  margin, not the pitch.

## 6. Roadmap

1. **Build #1 with the full system** (first real client or flagship concept piece):
   every gate, every agent, logged end-to-end. The system's own shakedown cruise.
2. **Retro → harden** whatever the shakedown exposes (supreme rule).
3. **Flagship portfolio piece** engineered for Awwwards submission (signature
   shader, TSL/WebGPU if verifiable).
4. **Research refresh ~2026-07-24** (6 weeks) — trend-scouts re-run, skills updated.
5. **Productization review** once 3 pieces exist: which building blocks become
   sellable components/templates.
