# web3d-concept — Creative Direction

Slop has no concept. Premium work sells an *idea* first, pixels second. No code until this phase produces a written concept.

## Why we failed here

Our past sites started from "what can the tool output" (tiles, a fake helix). Backwards. Start from a concept and let it dictate the tech.

## Process

1. **Interrogate the brief.** What does the client actually sell? Who is the audience? What's the ONE feeling the site must create (awe, calm, speed, luxury, playfulness)? What's the single business goal (book a call, buy, get funded)?
2. **Find the signature moment.** Every great 3D site has ONE thing people screenshot and share. A liquid-metal product reveal, a particle morph, a scroll that flies you through a world. Define it in one sentence. Everything else supports it. If you can't name it, you don't have a concept yet.
3. **Art direction.** Lock before building:
   - **Type system** — a display face + a text face, real scale (e.g. modular 1.25), tracking/leading rules. Type alone separates premium from slop.
   - **Color** — restrained palette (1 ink, 1–2 surfaces, 1 accent). Pull from the brand, not defaults.
   - **Material/mood** — matte vs glossy, organic vs technical, light vs dark. Write 3 adjectives and hold the whole site to them.
   - **Grid & whitespace** — premium sites are mostly empty space with confident type.
4. **Reference mining.** Pull 8–15 references (save links + what specifically to steal — a transition, a material, a layout rhythm). Sources: Awwwards (SOTD/SOTY), FWA, Codrops, Active Theory / Resn / Unseen / Locomotive / Aristide Benoist portfolios. Steal *techniques*, never designs.
5. **Storyboard the scroll.** Premium 3D sites are narratives told through scroll. Sketch section-by-section: what's on screen, what the camera/3D does, what the user feels. This becomes the build + motion spec.

## Output of this phase (write it to the project)

- One-paragraph concept + one-sentence signature moment.
- Art-direction sheet (type, color, mood adjectives, grid).
- Reference set with "what to steal" notes.
- Scroll storyboard (section list with 3D behavior per section).

Hand this to `web3d-assets` (what to model) and `web3d-build` (what to scaffold).
